## 差异清单

### 1) 页面范围差异
- 高保真：仅明确给出「首页/工作台」视觉稿（顶部栏、侧边栏、卡片区、公告区、帮助文档区）。
- 文档：要求完整路由覆盖 `login / dashboard / transfer / applications / approvals / transfers / users / settings / logs / profile / notifications / 403 / 404`。
- 处理：按文档补齐全部路由与页面占位，同时对 `dashboard` 按高保真做主还原。

### 2) 菜单项差异
- 高保真：侧边栏只明显展示「首页」「个人工作台」等少量菜单。
- 文档：要求角色化菜单（首页、申请单管理、审批管理、传输管理、用户管理、系统配置、日志审计）。
- 处理：菜单按文档角色规则实现动态渲染；高保真中未出现的菜单项按文档补齐。

### 3) 权限体系差异
- 高保真：未体现路由守卫、角色拒绝页、权限指令。
- 文档：要求 `requiresAuth/roles`、`/403` 跳转、`v-permission/v-role` 指令。
- 处理：实现全局守卫、403/404 页面、权限指令并在页面中使用示例。

### 4) 交互链路差异
- 高保真：偏静态展示，不含登录跳转、重定向、访问日志记录。
- 文档：要求登录重定向、未登录拦截、后置访问日志。
- 处理：补齐守卫逻辑，并在后置钩子写入通知 store 作为访问日志。

### 5) 布局切换机制差异
- 高保真：仅展示默认布局。
- 文档：要求默认布局 + 空白布局，并基于路由切换。
- 处理：实现 `DefaultLayout` + `BlankLayout`，通过 `meta.layout` 在 `App.vue` 动态切换。

### 6) 响应式规范差异
- 高保真：1920 视觉基准，未给具体断点代码。
- 文档：未明确断点实现细则。
- 处理：本次按移动端/平板/桌面断点补充 `clamp()` 与媒体查询，确保主布局可自适应。

---

## 本次实现结论
- `dashboard`：优先还原高保真视觉结构。
- 路由、权限、指令、403/404、多页面：按文档补齐。
- 若后续拿到更多页面高保真，可在不改动路由/权限骨架前提下继续替换页面细节。
