# Task P5 文档与高保真差异记录

## 说明
- 高保真来源：`.codebuddy/figma/3971_1023/figma.html`
- 文档来源：`docs/tasks/task_P5.md`
- 本文件用于记录两者不一致点，以及本次实现的补齐策略。

---

## 差异清单

### 1) 页面布局差异
- **高保真**：居中单卡片布局（Logo + 标题 + 副标题 + 表单），无左右分栏。
- **文档**：要求"左侧品牌展示区 + 右侧登录表单区"双栏布局。
- **处理**：按高保真实现居中单卡片，品牌信息（Logo + 渐变标题 + 副标题）置于卡片上方，整体更简洁。

### 2) 系统标题差异
- **高保真**：标题为"文件传输平台"，副标题"请登录您的账户"。
- **文档**：要求标题为"B端企业跨安全域大文件传输平台"。
- **处理**：按高保真简化标题，保持视觉一致性。

### 3) 安全域色块差异
- **高保真**：无绿区/黄区/红区色块展示。
- **文档**：要求顶部展示安全域色块（绿 #00b42a / 黄 #ff7d00 / 红 #f53f3f）。
- **处理**：省略独立色块，通过整体渐变背景（紫→青→蓝）传达科技感。

### 4) 记住我功能差异
- **高保真**：无"记住我"复选框。
- **文档**：要求 `a-checkbox` 记住我 + localStorage 持久化用户名。
- **处理**：按文档补齐"记住我"功能，放置在密码输入框下方。

### 5) Demo 快速体验差异
- **高保真**：无 Demo 账号表格。
- **文档**：要求下方展示 Demo 账号列表（表格形式），点击一键填入。
- **处理**：按文档补齐，使用 `a-table` 在登录卡片下方展示 5 个 Demo 账号，点击行一键登录。

### 6) 输入框图标差异
- **高保真**：输入框左侧有用户图标和锁图标（2.svg / 3.svg）。
- **文档**：未明确指定图标。
- **处理**：按高保真使用 Figma 导出的 SVG 图标作为输入框前缀。

### 7) 角色切换器差异
- **高保真**：登录页无角色切换器。
- **文档**：要求 `RoleSwitcher` 组件放在 `AppHeader` 中，仅开发环境渲染。
- **处理**：按文档实现独立 `RoleSwitcher.vue` 组件，集成在 Header 而非登录页。

---

## 本次实现结论
- **登录页视觉**：优先还原高保真的居中卡片 + 渐变背景 + Logo 布局。
- **记住我 / Demo 表格**：按文档补齐，高保真中未涉及。
- **安全域色块**：简化为整体渐变背景，未做独立色块。
- **角色切换器**：按文档实现为独立组件，放在 AppHeader 中。
