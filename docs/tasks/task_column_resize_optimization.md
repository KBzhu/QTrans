# 任务：优化表格列宽拖拽功能

## 任务背景

用户反馈当前表格列宽拖拽功能存在以下问题:
1. 表头列之间没有分隔符,用户不知道在哪里拖拽
2. 拖拽时鼠标轻微移动,列宽变化过大(可能存在宽度冲突)

## 任务目标

优化 `AssetDetectionTab` 组件的表格列宽拖拽体验,确保:
1. 列之间有明显的视觉分隔,提示用户可拖拽区域
2. 拖拽响应灵敏且准确,不会出现轻微移动导致剧烈变化

## 子任务

- [√] 1. 分析问题根源
  - [√] 检查当前 CSS 样式是否隐藏了拖拽手柄
  - [√] 检查列宽配置是否存在冲突(min-width/max-width vs width)

- [√] 2. 优化拖拽手柄样式
  - [√] 增加列分隔线的视觉样式(竖线或阴影)

  - [√] 移除 CSS 中的 min-width/max-width 约束

  - [√] 测试拖拽手柄是否明显可见

通过本次优化:

2. **解决了拖拽响应不准确问题**: 移除了 CSS 约束,避免与 columns 配置冲突
3. **提升了用户体验**: hover 和激活状态提供清晰的视觉反馈

优化后的表格列宽拖拽功能可以准确、流畅地响应用户操作,符合用户预期。

## 技术方案

### 问题1: 拖拽手柄不可见

**原因**: 当前样式中 `.arco-table-col-resize-handle` 背景设置为透明,且宽度仅 4px,用户难以发现

**解决方案**:
1. 增加列分隔线样式,使用伪元素或边框
2. 拖拽手柄增加默认背景色(浅灰色)
3. hover 时显示蓝色高亮和更宽的可点击区域
4. 拖拽时显示蓝色指示线

### 问题2: 拖拽响应不准确

**原因**: CSS 中定义了 `.col-file-name` 等类,设置了 `min-width` 和 `max-width`,与 columns 中的 `width` 冲突

**解决方案**:
1. 移除 CSS 中的宽度约束类(已有 columns 配置)
2. 确保所有列都使用 columns 的 width 属性
3. 为 resizable 列设置合理的初始宽度
