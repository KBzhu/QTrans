## 1. 项目概述

### 1.1 项目背景

企业内部存在多个安全域之间的数据传输需求，包括不同密级区域之间、内网与外网之间的文件交换。现有传输方式效率低、安全性不足、流程繁琐，亟需建设一套统一的大文件传输平台，实现跨安全域文件的安全、高效、可追溯传输。

### 1.2 项目目标

- 提供跨安全域的大文件传输能力，支持GB级别文件传输
- 实现完整的审批流程，确保传输过程合规可控
- 支持断点续传，保证传输可靠性
- 统一用户界面，提升用户体验
- 实现全流程追溯和审计

### 1.3 适用范围

本平台适用于企业内部员工、合作伙伴、供应商、子公司等用户群体，覆盖内网低密区域、高密区域、外网之间的文件传输场景。

## 3. 业务流程

### 3.1 核心业务流程

```
登录 → 创建传输申请单 → 上传文件 → 后台判断密级和加密上传 → 审批申请单 → 传输 → 归档下载
```


### 3.2 流程详细说明

#### 3.2.1 用户登录

**流程描述：**
1. 用户访问平台登录页面
2. 输入账号密码或通过SSO单点登录
3. 系统验证用户身份
4. 根据用户角色跳转至相应首页
5. 记录登录日志

**前置条件：**
- 用户账号已创建且状态正常
- 用户具有访问权限

**异常场景：**
- 账号不存在：提示"账号不存在"
- 密码错误：提示"密码错误"
- 账号被锁定：提示"账号已被锁定，请联系管理员"
- 无访问权限：提示"您暂无访问权限"

#### 3.2.2 创建传输申请单

**流程描述，以绿区传到绿区距离：**
1. 用户在首页选择传输类型，点击"绿区传到绿区"（选择传输类型）
2. 自动填充基本信息：
   - 自动生成申请人，生成申请单号，当前处理人，存储空间大小，上传有效期，下载有效期，上传区域和下载区域
3. 填写申请信息：
   - 部门-点击后弹出部门选择器
   - 数据传出国家/城市（点击后弹出城市选择器）
   - 数据传至国家/城市（点击后弹出城市选择器）
   - 下载人账号
   - 抄送人
   - 包含客户网络数据的radio  
   - 如果上面的选项为“是”，则必填以下字段：上传客户授权文件的上传框，SR单号，最小部门主管（自动带出）
   - 申请原因
   - 申请人通知选项
   - 下载人通知选项
4. 系统根据传输类型自动匹配审批流程
5. 点击“下一步”，校验页面必填的提交信息，提交申请单
6. 点击下一步成功后，申请单状态变更为"待上传文件"
7. 提交成功后，系统根据勾选的通知选项，发送通知给提交人
8. 页面自动进入上传文件页面

**业务规则：**
- 申请单号格式：`TRANS-YYYYMMDD-XXXXX`（如：TRANS-20260302-00001）
- 传输类型一经提交不可修改
- 用户选择的传出和传至城市，关系到数据到哪个就近的数据站点，所以需要填写
- 不同传输类型对应不同审批层级，例子中因为是低密到高密，所以免审批，如果是其他需要审批的传输类型，就需要增加文件密集筛选框，根据密集对应到底有多少级的审批
- 申请单号自动生成，不可编辑
- 用户如果提交中间退出页面，申请单将自动保存为草稿


#### 3.2.3 上传文件

**流程描述：**
1. 在申请单详情页点击"上传文件"
2. 选择本地文件或拖拽上传
3. 系统进行文件预检查：
   - 文件格式校验
   - 文件大小校验（单文件上限50GB）
   - 病毒扫描（可选配置）
4. 文件分片上传，支持断点续传
5. 上传完成后自动计算文件MD5
6. 后台判断文件密级，自动加密存储

**业务规则：**
- 支持多文件批量上传（单次最多100个文件）
- 单申请单大小上限：50GB
- 支持断点续传，上传中断后可继续上传
- 禁止上传文件类型：exe、bat、cmd、sh等可执行文件（可通过后台配置）
- 文件命名规则：不允许包含特殊字符（如：/ \ : * ? " < > |）

**断点续传机制：**
- 文件分片大小：5MB（可配置）
- 分片上传后记录上传进度
- 网络中断后自动检测已上传分片
- 支持手动恢复上传

#### 3.2.4 审批申请单

**流程描述：**
1. 审批人收到审批待办通知
2. 进入审批页面，查看申请单详情
3. 查看文件列表及预览（支持预览的格式）
4. 做出审批决定：
   - 通过：流转至下一审批节点或开始传输
   - 驳回：填写驳回原因，申请单退回提交人
   - 免审：特殊权限，直接通过所有审批环节
5. 记录审批意见和操作日志

**审批层级规则：**

| 传输类型 | 审批层级 | 审批顺序 |
|---------|---------|---------|
| 内网低密→高密 | 二级审批 | 一级审批人 → 二级审批人 |
| 内网高密→低密 | 三级审批 | 一级审批人 → 二级审批人 → 三级审批人 |
| 内网→外网 | 三级审批 | 一级审批人 → 二级审批人 → 三级审批人 |
| 外网→内网 | 二级审批 | 一级审批人 → 二级审批人 |

**驳回规则：**
- 驳回后申请单状态变更为"已驳回"
- 提交人可修改后重新提交
- 驳回原因必填，字数限制10-500字
- 驳回后需重新走完整审批流程

**免审规则：**
- 仅三级审批人和管理员具有免审权限
- 免审后跳过所有剩余审批节点
- 免审操作需填写免审原因，记录审计日志

#### 3.2.5 文件传输

**流程描述：**
1. 审批通过后，申请单状态变更为"待传输"
2. 后台调度传输任务
3. 系统执行文件传输：
   - 跨安全域传输
   - 传输过程加密
   - 传输进度实时更新
4. 传输完成后状态变更为"已完成"
5. 发送通知给提交人

**传输规则：**
- 传输优先级：特急 > 紧急 > 普通
- 支持传输限速配置
- 传输失败自动重试（最多3次）
- 传输失败后发送告警通知

#### 3.2.6 归档下载

**流程描述：**
1. 传输完成后，文件归档至目标域
2. 有权限的用户可下载归档文件
3. 记录下载日志（下载人、下载时间、下载IP等）

**下载规则：**
- 下载需二次身份验证（密码或短信验证码）
- 下载记录保存不少于6个月
- 支持批量下载（打包为zip）


| 传输 | /api/transfers/:id/retry | POST | 重试传输 |
| 日志 | /api/logs/operations | GET | 获取操作日志 |
| 通知 | /api/notifications | GET | 获取通知列表 |
| 通知 | /api/notifications/:id/read | PUT | 标记通知已读 |

### 7.2 接口规范

#### 7.2.1 统一响应格式

```typescript
interface ApiResponse<T> {
  code: number;        // 状态码：200成功，其他失败
  message: string;     // 提示信息
  data: T;            // 响应数据
  timestamp: number;   // 时间戳
}
```

#### 7.2.2 分页查询格式

```typescript
interface PageRequest {
  pageNum: number;     // 页码，从1开始
  pageSize: number;    // 每页条数
  sortBy?: string;     // 排序字段
  sortOrder?: 'asc' | 'desc';  // 排序方向
}

interface PageResponse<T> {
  list: T[];           // 数据列表
  total: number;       // 总条数
  pageNum: number;     // 当前页码
  pageSize: number;    // 每页条数
}
```

#### 9.2.2 命名规范

- 文件名：小写kebab-case，如 `user-list.vue`

- 常量名：大写SNAKE_CASE，如 `MAX_FILE_SIZE`
- CSS类名：BEM命名，如 `.user-list__item--active`

#### 9.2.3 TypeScript规范

- 严格模式，禁止使用 `any` 类型（特殊情况除外，需注释说明）
- 为所有接口定义类型
- 使用枚举定义常量
- 函数参数和返回值必须有类型注解

3. **图片优化**：使用合适的图片格式和压缩
4. **虚拟滚动**：长列表使用虚拟滚动
5. **防抖节流**：频繁操作使用防抖节流
6. **缓存策略**：合理使用浏览器缓存和Pinia持久化

### 9.4 测试要求

1. **单元测试**：核心工具函数和组件覆盖率 ≥ 70%
2. **集成测试**：关键业务流程测试
