## 1. 系统整体架构图

![系统架构图](placeholder)

**Mermaid 代码：**

```mermaid
%%{init: {'theme': 'base', 'themeVariables': { 'primaryColor': '#1890ff', 'primaryTextColor': '#fff', 'lineColor': '#8c8c8c'}}}%%
flowchart TB
    subgraph UI["用户交互层"]
        direction TB
        Browser[🌐 浏览器端]
        Mobile[📱 移动端]
    end

    subgraph Access["接入层"]
        direction TB
        Nginx["Nginx\n反向代理"]
        Gateway["API Gateway\n统一网关"]
        LB["Load Balancer\n负载均衡"]
    end

    subgraph Service["服务层"]
        direction TB
        subgraph Core["核心服务"]
            UserSvc["👤 用户服务"]
            AppSvc["📋 申请单服务"]
            FileSvc["📁 文件服务"]
            ApprovalSvc["✅ 审批服务"]
            TransferSvc["🔄 传输服务"]
        end
        subgraph Support["支撑服务"]
            NotifySvc["🔔 通知服务"]
            LogSvc["📊 日志服务"]
            ConfigSvc["⚙️ 配置服务"]
        end
    end

    subgraph Data["数据层"]
        direction TB
        MySQL[("🐬 MySQL\n主数据库")]
        Redis[("🔴 Redis\n缓存")]
        MinIO[("🗄️ MinIO/S3\n对象存储")]
        RabbitMQ["🐇 RabbitMQ\n消息队列"]
    end

    UI --> Access
    Nginx --> LB
    LB --> Gateway
    Gateway --> Service
    
    Service --> Data
    
    Service -. RPC .-> Service
    Service -. MQ .-> RabbitMQ

    style UI fill:#e6f7ff,stroke:#1890ff
    style Access fill:#fff7e6,stroke:#fa8c16
    style Service fill:#f6ffed,stroke:#52c41a
    style Data fill:#fff1f0,stroke:#f5222d
```

**说明**：展示系统五层架构，从用户交互层到数据层的完整请求链路，以及服务间的 RPC 和 MQ 通信。

## 3. 申请审批流程图

![申请审批流程图](placeholder)

**Mermaid 代码：**

```mermaid
%%{init: {'theme': 'base'}}%%
flowchart TD
    Start([📝 开始]) --> SelectType{选择传输类型}
    
    SelectType -->|绿区→绿区| AutoPass[✅ 自动通过]
    SelectType -->|绿区→黄区| L1[📤 一级审批]
    SelectType -->|绿区→红区| L2[📤📤 二级审批]
    SelectType -->|黄区→绿区| L2
    SelectType -->|红区→绿区| L3[📤📤📤 三级审批]
    SelectType -->|内网→外网| L3
    
    L1 -->|审批通过| Upload[📁 上传文件]
    L2 -->|审批通过| Upload
    L3 -->|审批通过| Upload
    
    AutoPass --> Upload
    Upload --> Check{文件校验}
    Check -->|通过| Queue[📋 加入传输队列]
    Check -->|失败| RejectFile[❌ 文件校验失败]
    
    Queue --> Transfer[🔄 开始传输]
    Transfer --> Monitor[📊 传输监控]
    
    Monitor -->|成功| Complete[✅ 传输完成]
    Monitor -->|失败| Retry{重试?}
    Retry -->|是| Transfer
    Retry -->|否| Failed[❌ 传输失败]
    
    L1 -.->|驳回| Reject1[📝 填写驳回原因]
    L2 -.->|驳回| Reject1
    L3 -.->|驳回| Reject1
    Reject1 --> Modify[✏️ 修改申请]
    Modify --> SelectType
    
    Complete --> Notify[🔔 通知下载]
    
    style Start fill:#1890ff,stroke:#1890ff,color:#fff
    style Complete fill:#52c41a,stroke:#52c41a,color:#fff
    style Failed fill:#f5222d,stroke:#f5222d,color:#fff
    style AutoPass fill:#52c41a,stroke:#52c41a,color:#fff
```

**说明**：展示从发起申请到传输完成的完整业务流程，包括不同传输类型的审批层级分支。

## 4. 文件分片传输时序图

![文件分片传输时序图](placeholder)

**Mermaid 代码：**

```mermaid
%%{init: {'theme': 'base'}}%%
sequenceDiagram
    participant Client as 客户端
    participant Gateway as API网关
    participant FileSvc as 文件服务
    participant MinIO as MinIO存储
    participant MQ as RabbitMQ
    participant Transfer as 传输服务

    Client->>Client: 1. 文件分片 (5MB/片)
    Client->>Gateway: 2. 请求上传凭证
    Gateway->>FileSvc: 3. 生成上传任务
    FileSvc->>MinIO: 4. 初始化分片上传
    MinIO-->>FileSvc: 5. 返回uploadId
    FileSvc-->>Gateway: 6. 返回上传URL
    Gateway-->>Client: 7. 返回上传凭证
    
    loop 分片上传
        Client->>FileSvc: 8. 上传分片
        FileSvc->>MinIO: 9. 存储分片
        MinIO-->>FileSvc: 10. 分片完成
        FileSvc-->>Client: 11. 返回分片结果
    end
    
    Client->>Gateway: 12. 合并分片请求
    Gateway->>FileSvc: 13. 触发合并
    FileSvc->>MinIO: 14. 合并文件
    MinIO-->>FileSvc: 15. 合并完成
    FileSvc->>MQ: 16. 发送传输任务
    FileSvc-->>Gateway: 17. 返回合并结果
    Gateway-->>Client: 18. 上传完成通知
    
    MQ-->>Transfer: 19. 消费传输任务
    Transfer->>Transfer: 20. 执行跨域传输
    Transfer-->>MQ: 21. 传输完成回调
```

**说明**：展示文件从分片上传、合并、到触发跨域传输的完整时序。

## 使用说明

### 嵌入到 Markdown 文档

1. 将上述 Mermaid 代码块直接复制到你的设计说明书 `.md` 文件中
2. 确保 Markdown 渲染器支持 Mermaid（如 GitHub、GitLab、飞书、语雀等）
3. 图片 placeholder 处可删除，不影响渲染

### 导出为图片

1. 访问 [Mermaid Live Editor](https://mermaid.live)
2. 粘贴 Mermaid 代码
3. 点击右上角 "Actions" → "PNG" 或 "SVG" 导出

### 调整样式

- 修改 `themeVariables` 中的颜色值可自定义配色
- 可选主题：`default`、`base`、`neutral`、`forest`、`dark`
