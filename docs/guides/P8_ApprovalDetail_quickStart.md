# P8.2 审批详情页面快速上手

> 目标：5~10 分钟内跑通审批详情页面，完成信息查看、文件下载、审批时间线展示与审批操作验证。

## 1. 入口与运行

- 页面路由：`/approvals/:id`
- 路由配置：`qtrans-frontend/src/router/routes.ts`
- 页面文件：`qtrans-frontend/src/views/approvals/ApprovalDetailView.vue`

本地启动：

```bash
npm --prefix d:\VibeCoding\QTrans-0302new\qtrans-frontend run dev
```

示例地址：`http://localhost:5173/approvals/app-1002`

---

## 2. 页面结构

1. 顶部：申请单号、当前审批层级、状态标签
2. 顶部审批操作（待审批态）：通过 / 驳回 / 免审（管理员）
3. 双 Tab：
   - `申请单信息`：基本信息、申请信息、审批时间线
   - `文件列表`：文件表格 + 单个/批量下载
4. 底部审批意见区：输入审批意见并执行动作

---

## 3. 复用与抽象

## 5. 给测试看的回归用例（QA）

### 5.1 测试前置

- 开启 Mock：`VITE_MOCK_ENABLED=true`
- 使用有审批数据的申请单（如 `app-1002`）

### 5.2 功能回归路径

1. 页面进入
   - 操作：访问 `/approvals/app-1002`
   - 预期：页面完整展示，无报错

2. Tab 切换
   - 操作：切换「申请单信息」与「文件列表」
   - 预期：区域切换正常，数据不丢失

3. 时间线展示
   - 操作：查看审批时间线
   - 预期：含提交节点、审批节点，审批意见可展开

4. 审批通过
   - 操作：填写意见并点击“通过”
   - 预期：出现确认框，确认后提示成功，状态更新

5. 审批驳回
   - 操作：不填意见点击“驳回”，再填写意见后驳回
   - 预期：首次被拦截提示；填写后可正常驳回

6. 免审（管理员）
   - 操作：管理员角色点击“免审”并确认
   - 预期：提示成功，状态更新为通过

7. 文件下载
   - 操作：文件列表执行单个下载与批量下载
   - 预期：下载动作触发并有提示

### 5.3 边界验证

- 非法申请单 ID：页面无崩溃，有空态/兜底显示
- 已结束状态（已通过/已驳回）：操作按钮自动隐藏
- 连续切换 Tab + 审批动作：页面状态一致
